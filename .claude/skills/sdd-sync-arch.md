# SDD 架构文档同步

## 触发

用户说 `/sdd-sync-arch` 或「架构变了，更新一下文档」

## 执行步骤

1. 检查当前 diff 中涉及架构层面的变更：
   - 新增/删除模块
   - 通信协议变更
   - 数据库模型变更
   - 目录结构变更
   - 新增/替换外部依赖
2. 读取 `docs/ARCHITECTURE.md`
3. 定位需要更新的章节
4. 原地更新（不追加，直接替换为当前真貌）
5. 检查是否需要新增 ADR（如果有架构决策变更）
6. 如需新增 ADR，生成 `docs/adr/NNN-<title>.md` 骨架
7. 输出变更摘要

## 约束

- 严格遵循「原地更新到当前真貌」的演进规则
- 不保留旧版本内容
- 如有重大架构决策变更，必须同步创建 ADR
