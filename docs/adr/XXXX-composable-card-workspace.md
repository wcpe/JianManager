- **上下文**: ADR-030 把实例工作区定为「可分屏面板（`react-resizable-panels`）」，落地 FR 为 FR-129（分屏）/ FR-130（文件+配置合并），**二者均未实现**。前端整体重设计（design §9）提出更强的方向：工作区不是固定分屏，而是**可拖拽的卡片画布**——卡片 = 实例 × 功能，自由摆放 / 调大小 / 存预设，并作为 FR-167（跨实例超级工作台：多实例卡片拼监看墙）与 FR-168（导播台：多预设预热瞬切）的前置底座。固定分屏（左右两栏拖比例）撑不起「任意拼卡 + 跨实例 + 预设轮播」。用户已确认采用画布方向、取代 ADR-030 并并入 FR-129/130。

## 决策

**实例工作区从 ADR-030 的「固定可分屏面板」改为「可组合卡片画布」，取代 ADR-030。**

1. **拖拽网格画布**：用 `react-grid-layout`（或等价 `gridstack`）替代 `react-resizable-panels`。卡片在网格里自由摆放、调大小、流式不重叠。
2. **卡片 = 实例 × 功能**：终端 / 资源（文件+配置合一，**承 FR-130**）/ 插件 / 监控 / Bot / 状态各为一卡；卡头 = grip 拖拽 + 全屏 + 关闭。
3. **预设个人级持久化**：命名保存画布布局（先 `localStorage`）；原固定 Tab 降级为「快捷预设」（默认「运维台」布局），新手默认、高手自由拼。
4. **状态外提 + 惰性挂载**（承 ADR-030 这两点正确决策）：画布维护卡片树（布局 + 各卡资源 + 实例），各卡自管 dirty；非激活/未挂载卡不预渲染、不建 WS。
5. **作用域分层**：本 ADR + FR-166 限**单实例画布**；跨实例（FR-167）与导播台多 WS 预热并发（FR-168）在此画布上叠加，各自另立 FR（FR-168 的并发上限/降频另写 ADR）。

## 理由
- 画布比固定分屏灵活，直接兑现 design §9「超越固定 tab/分屏」，且是 167/168 唯一可行底座（固定两栏无法拼跨实例监看墙）。
- 保留 ADR-030 的「状态外提 + 惰性挂载」（这两点与画布同样需要），只换「固定分屏 → 拖拽画布」的布局机制与依赖。
- 文件+配置合一（FR-130）在资源卡内自然实现，认知负担与实现成本都低。
