# ADR-009: 运维控制台为主 Shell

- **日期**: 2026-06-19
- **状态**: accepted
- **上下文**: 现有主布局（DashboardPage）是单一导航侧栏 + 路由内容区；实例终端埋在实例详情页的 Tabs 里；Bot 只有一个独立 `/bots` 列表页。运维一个多实例、多 Bot 的 MC 群组服时，要在「实例列表 → 详情页 → 终端 Tab」与「/bots 页」之间反复跳转，缺少一个「边看终端边管实例与 Bot」的统一工作台。压测场景下单实例 Bot 可达上万（见 FR-009 容量），现有扁平列表撑不住。
- **决策**:
  1. **以「运维控制台」三段式侧栏取代现有主布局**：上=功能导航（仪表盘/节点/实例/Bot/告警…）、中=节点切换下拉 + 实例树（常驻）、下=系统平台导航（用户/审计/设置/退出）、右=工作区。登录默认进控制台。
  2. **实例树常驻 + 点实例在工作区开终端**：节点下拉切换（全部/某节点），实例树显示状态点；点实例在右侧工作区开其终端（复用一次性 token + xterm）。分屏/分组/导播台为后续阶段。
  3. **Bot 双表面**：① 控制台工作区内每实例的 Bot 段（`终端 | Bot` 切换）；② 全局 `/bots` 页（跨实例总览，导航位于「实例」与「告警」之间）。两者同源数据。
  4. **Bot UI 一律「聚合优先、永不全量铺开」**：实例树内只挂聚合徽标，工作区/全局页用概览卡片 + 分组 + 分页 + 批量；上万 Bot 不逐行渲染。
  5. **保留全部既有导航项**，仅按布局重组（不照搬预览 HTML 的占位项），控件用项目自有 shadcn/ui。
- **理由**:
  - 统一运维入口，消除「列表/详情/终端/Bot 页」间的反复跳转，实例-终端-Bot 一处协同。
  - 复用现有终端/实例/节点/Bot API，不新增进程边界、不违反架构不变量。
  - 聚合优先是 Bot 压测规模（~12,800）下唯一可行的呈现方式。
- **后果**:
  - DashboardPage 主布局重构（FR-037）；登录落地页由概览改为控制台。
  - 实例树常驻 + Bot 聚合徽标需要聚合摘要 API（FR-038 `GET /bots/summary`），否则大列表/计数会拉全量。
  - 关联 FR-037（布局）、FR-039（控制台 Bot 段）、FR-040（全局 Bot 页）。
- **替代方案**:
  - 维持现状仅做样式美化 — 解决不了跳转与协同，否决。
  - 页面内二级侧栏（保留全局侧栏 + 内容区再加实例树）— 双侧栏 chrome 冗余（约 480px），否决。
  - 所有页面全局换三段式 — 非实例页（用户/审计…）不需要实例树，不合理，否决。
