# ADR-030: 实例工作区可分屏面板架构

- **日期**: 2026-06-24
- **状态**: **superseded-by ADR-034**（可组合卡片工作区）——固定可分屏面板方向被「可拖拽卡片画布」取代；FR-129/130 并入 FR-166。保留本 ADR 的「状态外提 + 惰性挂载」决策（ADR-034 沿用）。
- **上下文**: ADR-009 立「运维控制台」时把「分屏 / 分组 / 导播台」明确列为**后续阶段**（FR-037 验收里「分屏」「切导播台」为禁用占位）。当前实例详情是固定的六个 Tab（终端/文件/配置/插件/备份/Bot），一次只看一个；且「文件」与「配置」高度同源（`ConfigExplorer` 仅是 `ResourceExplorer` 的薄封装，同数据源 `@/api/files`、同目录树/工具栏，仅打开文件时多注入配置编辑器）。运维一个实例时常要「边看终端边改配置 / 边看文件边看监控」，固定单 Tab 切来切去、且文件/配置割裂成两个 Tab 是体验瓶颈。工作区状态（`openFile`/`dirty`/`selection`/`clipboard`）目前全挤在 `ResourceExplorer` 单体与 `console.ts` store，固定布局 `h-[600px]`/`w-1/2`/`w-56` 与可拖拽分屏冲突。

## 决策

**实例工作区从「固定单 Tab」演进为「可分屏的面板（panel）工作区」，兑现 ADR-009 的分屏占位。**

1. **面板类型化**：把现有 props 驱动的子组件抽成可复用「面板类型」——终端 / 资源（文件+配置合一）/ 插件 / 监控（含探针与历史曲线）/ Bot。面板各自承载一类能力。
2. **文件与配置合并为「资源」面板**：删除割裂的「文件」「配置」双 Tab，合为一个资源面板（配置发现/收藏作为该面板的可选视图）。因 ConfigExplorer 已是 ResourceExplorer 薄封装，合并近零成本（FR-130）。
3. **可分屏 + 可调比例**：用 `react-resizable-panels`（或等价）替换固定栅格，面板可水平/垂直拆分、比例可拖拽；支持同时打开多个面板（如左终端右配置）。多文件以面板内 Tab 承载。
4. **状态外提**：工作区维护一棵 panel 树（哪些面板、布局、各面板打开的资源），**各面板自管 dirty / 未保存草稿**；不再由单体 ResourceExplorer 一锅端。
5. **惰性挂载**：非激活/未打开的面板不预渲染（现状六 Tab 全挂载会同时建 WS、发 discover）。

## 理由
- 兑现 ADR-009 既定方向，不引入新进程边界、不违反架构不变量（仍是浏览器↔Worker 既有 WS/HTTP）。
- 文件/配置合并消除重复 Tab 与认知负担，且实现成本极低。
- 面板化 + 分屏让「终端+配置+监控」一处协同，契合多实例运维。
- 状态外提是分屏的前置：单体内嵌的 openFile/dirty 无法支撑多面板并存。

## 后果
- `ResourceExplorer` 退化为「文件树 + 打开调度器」，编辑/保存/dirty 下沉到面板。
- 新增工作区 panel 树状态（复用 Zustand），与 ADR-031 的「工作区可寻址」协同（打开的面板/资源可进 URL 以支持刷新/分享）。
- 详情页「备份」「Bot」等仍以面板或区段形式保留；六 Tab 布局被面板工作区取代。

## 关系
- **ADR-009（运维控制台为主 Shell）**：本 ADR 兑现其「分屏为后续阶段」的占位，是其延续而非推翻。
- **ADR-031（导航/视图状态可寻址化）**：面板树的可寻址（打开哪些面板/资源进 URL）由 ADR-031 承载。
- **FR-129（工作区分屏面板化）/ FR-130（文件+配置合并）**：本 ADR 的落地 FR。
