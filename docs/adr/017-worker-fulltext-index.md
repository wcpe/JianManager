# ADR-017: Worker 本地持久全文索引

- **日期**: 2026-06-23
- **状态**: accepted
- **上下文**: FR-074 要求对实例工作目录做跨文件全文搜索（关键字 → 命中文件 + 行号 + 片段）与文件名快速打开（quick-open）。大目录（成百上千文件、动辄数百 MB）下若每次搜索都现场全量遍历正则扫描，响应不可接受，故需要**持久化倒排索引**并随文件变更**增量更新**。架构红线（见 `architecture-invariants.md`）明确：文件归 Worker、**Worker 不得直接访问 CP 数据库**。因此索引必须是 **Worker 本地资产**，落进节点数据根（ADR-010）的 `var/index/`，经既有 gRPC（ADR-002）被 CP 查询委托，绝不进 CP 的 SQLite/MySQL。需确定索引引擎与落位、增量策略、忽略规则。

## 决策

### 1. 索引引擎：在树内最小倒排索引（拒绝 bleve 与 SQLite FTS5）

候选与取舍（题面要求 bleve vs SQLite FTS5 二选一，下记真实评估结论）：

- **SQLite FTS5（Worker 本地库）—— 否决**：本项目 SQLite 驱动是 `github.com/glebarez/sqlite`（底层 `modernc.org/sqlite`，纯 Go、无 cgo）。modernc 的 SQLite **不编译 FTS5 扩展**，要用 FTS5 必须换成 `mattn/go-sqlite3` 并启用 `-tags "sqlite_fts5"` + cgo。引入 cgo 直接破坏「Go 单二进制、跨平台交叉编译、无 C 工具链依赖」的部署前提（ADR-001/005 的精神），代价过高。
- **bleve（纯 Go 嵌入式）—— 否决**：bleve 确是纯 Go、可嵌入，功能上能满足。但它为通用搜索（分析器管线、多种 query、scorch 段存储、评分排序）设计，依赖树庞大（`blevesearch/*` 一系列模块 + RoaringBitmap + protobuf 段格式等），而 FR-074 的实际需求只是「子串/词项 → 命中文件 + 行号 + 片段」的运维向 grep，不需要相关性评分、不需要复杂查询语言。为此引入十余个传递依赖与其维护面，违背项目最小依赖、便携自包含的取向。
- **决策 ⇒ 在树内自实现最小倒排索引**：在 `internal/worker/search` 写一个**纯 Go、零新增依赖**的倒排索引——把每个文本文件按 token（小写化的 `[a-z0-9_]+` 词元 + 可选 n-gram 兜底子串）映射到「文件 → 行号集合」，配合一张「文件路径 → 指纹（size+mtime）」表做增量。查询时倒排取候选文件，再在候选文件内做一次精确行扫描提取行号与片段（候选集远小于全量，故可接受）。文件名 quick-open 用一张「basename/路径 → 文件」的轻量结构做子串匹配，无需倒排。

> 这是对题面「bleve / FTS5 二选一」的诚实回应：二者按本项目约束分别因 **cgo 依赖**（FTS5）与 **依赖树过重**（bleve）被否，故选与之对等、但零依赖的在树实现。倒排+持久化+增量+忽略规则的核心语义不变。

### 2. 落位：节点数据根 `var/index/`（Worker 本地资产，不碰 CP DB）

- 索引根：`<dataRoot>/var/index/<instance-uuid>/`（数据根见 ADR-010；`var/index` 为本 ADR 新增的 FHS 子目录，随数据根整体迁移）。
- 每实例一份独立索引（gob 编码落盘：`index.gob` 倒排 + 文件指纹表）。一个实例的索引损坏/删除不影响其它实例；删除实例时连带删除其索引目录。
- **绝不写入 CP 数据库**：索引是 Worker 私有的派生数据，CP 仅经 gRPC `SearchFiles` 拿查询结果，不持有索引本身。符合「数据库仅 CP 可读写 / 本地实例数据仅 Worker 可读写」不变量。

### 3. 构建与增量更新策略

- **首建（惰性）**：某实例首次被搜索（或 quick-open）时，若无落盘索引则全量构建一次并落盘。构建在 Worker 进程内同步进行，带文件数/总字节上限保护（见忽略规则与限额）。
- **增量**：每次查询前做一次轻量「再校验」——遍历工作目录当前文件的指纹（size+mtime），与索引内指纹表比对：新增/变化文件重新索引、删除文件移除其倒排贡献，未变文件跳过。变更后落盘。增量以「目录扫描 + 指纹比对」驱动，**不依赖文件系统事件监听**（跨平台一致、Worker 重启后无状态丢失），代价是查询前的一次 stat 遍历，对运维向搜索可接受。
- **并发**：每实例索引有自己的锁，构建/增量/查询串行化到该实例；不同实例并行。

### 4. 忽略规则（glob，可配）

- 默认忽略：常见非源码/二进制/运行态目录与扩展名——`logs/`、`cache/`、`crash-reports/`、`.git/`、`node_modules/`、`world*/`（MC 世界数据，海量二进制）、以及 `*.jar`、`*.zip`、`*.gz`、`*.png`、`*.jpg`、`*.dat`、`*.mca`、`*.lock` 等二进制/归档扩展。
- **二进制探测**：对未被扩展名规则排除的文件，读取首部若含 NUL 字节或非 UTF-8 比例过高则判为二进制并跳过（避免把二进制当文本塞进倒排）。
- **体积/数量上限**：单文件超过阈值（默认 2 MiB）只索引文件名不索引内容；实例索引文件数上限（默认 50000）防爆。
- **可配**：忽略 glob 经 Worker 配置 `search.ignore`（`worker.yaml` / `JIANMANAGER_SEARCH_IGNORE`，与 ADR-020 配置惯例一致）追加到默认集；默认集内置、零配置可用。

### 5. 查询路径（gRPC，CP 仅转发）

- 新增 gRPC `SearchFiles(SearchFilesRequest) returns (SearchFilesResponse)`：入参 `instance_uuid` + `query` + `mode`（content / filename）+ `max_results`；返回命中列表（每项：相对路径 + 行号 + 行片段；filename 模式行号为 0）。
- CP 新增 `POST /instances/:id/files/search` 把请求经既有 `ClientPool` 转发到目标节点 Worker 的 `SearchFiles`，鉴权复用文件读权限（`canAccessInstance`）。CP 不缓存、不落库索引，纯转发。

## 理由

- **守架构红线**：索引是 Worker 本地派生资产，落数据根 `var/index/`，CP 只经 gRPC 取结果——「Worker 不碰 CP DB」「文件归 Worker」「gRPC 唯一 RPC」三条不变量全部满足。
- **零新增依赖、单二进制**：在树倒排避免 cgo（FTS5）与重依赖树（bleve），与项目便携自包含、跨平台交叉编译的取向一致。
- **够用即可（YAGNI）**：运维向「跨文件 grep + 跳行」不需要相关性评分/复杂查询，最小倒排 + 候选内精确扫描已满足验收（命中文件+行+片段、增量、忽略规则）。
- **增量经指纹比对而非 fs 事件**：跨平台一致、无监听器资源占用、Worker 重启即自愈，代价（查询前一次 stat 遍历）可控。

## 后果

- 数据根新增 `var/index/` 子目录（ADR-010 布局的加性扩展），随数据根整体迁移；删实例时需清理对应索引目录。
- 索引为派生数据：可随时删除重建，不需备份（备份 FR-013/056 应排除 `var/index`，与排除 `cache/` 同理）。
- 最小倒排不做词干/同义/模糊与相关性排序；若未来需要这些，再评估引入 bleve（届时以新 ADR 取代本节引擎决策）。
- 超大文件只索引文件名、二进制跳过：这些文件的**内容**搜不到（符合预期，运维向）。
- 增量依赖 mtime/size 指纹：极端情况下「同尺寸同 mtime 但内容变化」会漏更新（概率极低；可经强制重建规避，留作已知取舍）。

## 替代方案

- **SQLite FTS5（Worker 本地库）**：纯 Go 驱动不带 FTS5，启用需 cgo + `mattn/go-sqlite3`，破坏单二进制/交叉编译；否决（见决策 1）。
- **bleve 纯 Go 嵌入式**：功能满足但依赖树过重、为通用搜索设计、超出 grep 式需求；否决（见决策 1）。
- **每次现场全量正则扫描（无索引）**：大目录响应不可接受，违背 FR-074「大目录响应可接受」；否决。
- **把索引塞进 CP 数据库集中存储**：违反「Worker 不碰 CP DB」与「文件归 Worker」不变量；否决。
- **基于 fsnotify 的实时增量**：跨平台行为不一、监听器资源占用、Worker 重启丢状态需重建；改用查询前指纹比对，更简单自愈。

## 关系

- **ADR-002（gRPC 节点通信）**：搜索查询经新增 gRPC `SearchFiles` 委托，CP↔Worker 仍只用 gRPC。
- **ADR-010（便携数据根）**：索引落 `var/index/`，是该布局的加性子目录，随数据根迁移。
- **FR-070（资源管理器）**：搜索面板挂进 FR-070 的 `ResourceExplorer`，命中点击复用其「按路径打开 + 编辑器」通路（编辑器定位到行）。
