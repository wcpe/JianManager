# ADR-004: 用户组替代多租户

- **日期**: 2025-06-16
- **状态**: accepted
- **上下文**: 系统需要支持多个独立团队/客户使用同一平台。全量多租户（tenant_id 隔离）需要每张表加 tenant_id、每个查询加过滤、中间件注入上下文，给每个功能模块增加 30-50% 开发量。单人开发难以承受。
- **决策**: V1 采用「用户组 + 服务分配」模型，不做全量多租户。
- **理由**:
  - 用户组（Group）+ 实例分配（GroupInstance）已满足数据隔离需求
  - 组配额（GroupQuota）满足资源限制需求
  - 平台管理员看全部，组管理员看组内，组成员看分配的实例
  - 开发量减少 30%+
  - MCSManager 管理几千实例也没有多租户
- **后果**:
  - 如果未来需要真正多租户（如 SaaS 平台），需要在 Group 模型上加 tenant_id
  - 跨组查询需要管理员权限
- **替代方案**:
  - 全量多租户（tenant_id 隔离）— 功能完整但开发成本过高
