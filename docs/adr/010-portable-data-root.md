# ADR-010: 项目自包含便携运行时目录布局

- **日期**: 2026-06-19
- **状态**: accepted
- **上下文**: 当前运行态路径散落——`servers_dir` 配在 worker.yaml（任意相对/绝对路径），JDK 硬编码装在 `<serversDir>/jdks`，实例 `work_dir` 由用户手填绝对路径，无统一数据根。运维方希望「项目即一个自包含目录」：JDK、配置、所有服务器、下载的核心都在内，可整体迁移，目录用 bin/etc 等标准名。
- **决策**:
  1. **单一项目内数据根**：默认项目内 `./data`，可经 `JIANMANAGER_DATA_DIR` 覆盖；进程启动时若不存在按布局自动初始化。
  2. **FHS 式标准布局**：`bin/`（平台/辅助可执行）、`etc/`（平台与节点配置）、`opt/jdks/<vendor>-<ver>/`（便携 JDK）、`var/servers/<slug>-<shortid>/`（系统分配的服务器工作目录）、`var/log/`、`var/artifacts/`（制品库，见 ADR-011）、`cache/`（临时：下载中转/解压）。
  3. **一切收口到根、按根相对解析**：JDK 装入 `opt/jdks`（取代 `<serversDir>/jdks`）；实例工作目录由系统在 `var/servers` 下按 slug+shortid 分配（取代用户手填）；登记路径相对数据根存储，数据根整体拷走后仍自洽。
- **理由**:
  - 自包含 + 便携是中小运营方「拷贝即迁移、删除即清理」的硬需求。
  - 统一根消除散落路径与「绝对路径写死」导致的不可移植与 BUG-004 类问题。
  - 标准目录名让人工排查、备份、归档都有据可依。
- **后果**:
  - 细化并先于 FR-032（工作目录分配）/ FR-033（JDK 位置）；FR-033 已实现的 `<serversDir>/jdks` 需迁到 `opt/jdks`。
  - 引入数据根配置项（CP 与 Worker 同源约定）；FR-044 负责初始化与接线。
  - 细化 ADR-007（工作目录系统分配 → 落 `var/servers`）与 ADR-008（托管 JDK → 落 `opt/jdks`），二者决策不变，仅明确落位。
- **替代方案**:
  - 维持散落路径 + 用户填绝对路径 — 不可移植、易错，否决。
  - 装系统级 JDK / 全局目录 — 与便携自包含相悖，否决。
