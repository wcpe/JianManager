# FR-177 spec：节点管理页重做（方案 A 主从双栏）

> 状态：待审　·　关联 PRD：FR-177（增强 FR-144）　·　分支：feature/fr-177-node-page　·　依赖：FR-178（JDK/缓存面板）、BUG-A（坏节点修复后端，已落）、FR-176（卡片基线，已落）

## 1. 背景与目标

现 `NodesPage` 为卡片网格/列表 + kebab 菜单 + 手搓 `fixed inset-0` 模态（JDK/端口），用户评"太简陋、流程稀烂"。FR-144 交付版仍不够直观、信息密度低。

**目标**：按用户选定的**方案 A 主从双栏**重做——左节点列表（可收缩）+ 右实时详情（分段承载 JDK/制品缓存/端口/监控/坏节点修复），高信息密度、切换不丢上下文，与群组管理页（FR-145）主从双栏同构、风格统一。

## 2. 范围

### 范围内
- 左栏：集群汇总头 + 搜索 + 添加节点 + 节点列表（状态/名/host/mini 水位/实例数），**可收缩为窄图标轨**。
- 右栏：选中节点详情（身份/操作 + 资源仪表 + 分段 Tabs），承载 FR-178 的 JDK 面板 + 制品缓存面板 + 端口 + 监控曲线 + **坏节点修复入口**（BUG-A 的 `/nodes/repair/*`、`/nodes/:id/reenroll|orphans|purge-orphans`，ADR-039 §2 已有后端）。
- 抽屉 UX 约束沿用（FR-178 §5：分段不重组、布局不跳）。

### 非目标
- 不做 JDK/缓存面板本身（FR-178）；不改节点 CRUD/维护/排空/下线后端（已有）；不改坏节点修复后端（BUG-A 已落，本 FR 只加 UI 入口）。

## 3. 设计

### 3.1 整体布局（`web/src/pages/NodesPage.tsx` 重写）
- 主从双栏：`左列表（可收缩） | 右详情`。无选中→右栏空态「选择左侧节点查看详情」。
- 左栏宽（默认 ~280px）⇄ 收缩（窄轨 ~56px，仅状态点 + 名首字/图标，hover tooltip 显名，点展开）；收缩态持久（console store / localStorage）。

### 3.2 左栏
- 顶：集群汇总头（在线/离线/维护计数 chip + CPU/内存/磁盘聚合水位，复用 `summarizeNodes`）+ 搜索框 + 「添加节点」按钮（复用 `AddNodeDialog`）。
- 列表：每行 = 状态点（呼吸灯运行态）+ 名 + host + mini 水位条（CPU/内存，复用 `MiniBar`）+ 实例数；选中高亮；离线置灰。复用/精简 `NodeWorktableCard` 的资源逻辑为列表行。

### 3.3 右栏（选中节点详情）
- 身份块：图标 + 名 + host + 系统/架构 + 状态徽标 + 操作 kebab（进入维护/排空/下线，复用现有 `NodesPage` 的动作 + `DangerConfirm`）。
- 资源仪表：CPU/内存/磁盘/负载环（复用 `ResourceGauge`）。
- 分段 Tabs（抽屉 UX 约束，切段不重组）：
  - 概览（IP/系统/网络/端口，复用 `NodeOverviewSection`）
  - 实例（该节点实例列表 + 各实例指标对比，复用 `NodeInstanceCompare`）
  - JDK（FR-178 JDK 面板）
  - 制品缓存（FR-178 缓存面板）
  - 端口（复用 `NodePortsPanel`）
  - 监控（历史曲线，复用 `NodeDetail` 的 TimeSeriesChart 组）
  - 坏节点修复（BUG-A 入口：疑似坏节点诊断 + 重 enroll + 清理孤儿，破坏性走 `DangerConfirm`）

### 3.4 复用与去除
- 去除手搓 `fixed inset-0` JDK/端口模态（506-531），改右栏分段（JDK/端口）+ 抽屉（FR-178）。
- 复用既有 `NodeDetail`/`NodeOverviewSection`/`NodeInstanceCompare`/`ResourceGauge`/`summarizeNodes`/`StatusBadge` 等，避免重写。
- 卡片网格视图：方案 A 以主从双栏为主，**不保留**原卡片网格（左栏列表即概览）；如需可后续加密集列表切换（非本 FR）。
