# 前端重设计 · 设计记录

> 状态:**设计探索完成**(2026-06-26,全部页面已定稿,见 §10 + 可交互原型 `preview.html`)。本文档驱动 FR 拆分与实现。
> 原则提醒:视觉要精致,但**信息与结构保持简单,不过度设计**。

## 1. 设计原则

### 1.1 视觉语言(靛蓝圆角精致风)
- 简约、扁平、舒适;**整体大量运用圆角**:组件圆角、卡片大圆角(~16px)、标签/分段做成小巧紧凑的**圆角 pill**。
- 卡片用**柔和弱阴影**分层,而非硬边框一圈圈框。
- **悬浮反馈 + iOS 丝滑**:hover 微交互(卡片轻抬、行高亮、标签变色),过渡用 iOS 那种减速缓动 `cubic-bezier(.16,1,.3,1)`;运行对象用**呼吸灯**(脉动光环)表达"活着"。
- 配色**不纯黑纯白、不死灰**:文字深石板灰 `#202533`、背景带色温浅灰 `#F4F5F7`、柔和语义色(绿 `#10B981`/琥珀 `#F59E0B`/红 `#EF4444`)。
- 间距舒适、留白大方。

### 1.2 主题(全局)
- **全局双主题**:靛蓝 `#6366F1`(默认)/ 青绿 `#14B8A6`(备选)。
- **全站统一,一处切换全站生效**,`localStorage` 持久,与明暗模式正交。
- 主色全由 CSS 变量驱动;切换时曲线/图标/标签/进度条跟主色变,结构色不变。

### 1.3 信息/布局原则
- 卡片提质:图标 + 数字层次 + 信息**成组左对齐**;KPI 右侧**按指标混搭**(占比→环/长条、走势→趋势线、计数→双值)。
- 列表**不平铺**:大规模用**多级嵌套分组 + 折叠优先**。
- 统一卡片原语(弃 shadcn `Card` 松散用法,统一紧凑 `Panel`)。

## 4. 已定页面设计

### 4.1 视觉规范 + 卡片
- 见 §1。卡片:图标块(语义/主色)+ 数字层次 + 趋势/`MiniBar`;右侧按指标混搭。

### 4.2 监控页骨架(平台/节点/实例共用)
- 6 指标:资源使用率、负载(1/5/15 分)、CPU、内存、磁盘 IO、网络 IO。
- 每图独立时间筛选(昨天/今天/最近七天/自定义)+ 底部 **brush 拖拽轴**。
- **hover 浮窗**:该时刻各指标值 + **进程 TOP10**(PID/进程名/占用/启动用户/cmd)。
- 底部「当前时间点 · 进程明细」:按 CPU/内存/磁盘/网络 切换查任意时间点。
- **监控(历史)+ 实时** 并存。换指标:节点监控复用 `NodeInstanceCompare`;实例监控 = TPS/MSPT/堆/线程/玩家/区块。
- 视觉为靛蓝圆角风 + 双主题切换(参考宝塔监控交互,但不照抄其朴素视觉)。

### 4.3 平台信息页(原"主机",已改名)
- 资源卡(CPU/内存/磁盘/负载)+ 主机硬件(主机名/IP/OS/核数/磁盘/时区) + 平台运行时(版本/PID/启动/最近重启/数据库/HTTP·gRPC 端口/协程/Go GC)。

### 4.4 实例多级分组完整页 = 左树 + 右列表
- **左** = 分组树:新建组 / 嵌套子组 / 折叠优先(折叠只渲染分组头)/ 选中,节点挂计数。
- **右** = **复用 §4.5 列表骨架**显示选中组的实例 + 面包屑(组路径) + 批量「标记入组」。
- 后端:分组树(自引用 `parent_id` + 实例-组 M:N)。

### 4.5 列表骨架 = 一套骨架,长出两种形态

**关键认知:不是"一个模子套十几页"。** 统一的是**视觉语言 + 交互骨架**(靛蓝圆角、灵动 hover、iOS 丝滑、筛选/搜索/批量/分页、状态 pill、双主题);在这之上按对象本质分两类:

| 类型 | 对象 | 默认视图 | 卡/行里放什么 | 特有交互 |
|---|---|---|---|---|
| **运行实体** | 实例 / 节点 / Bot | 卡片(可切列表) | **内嵌资源**(CPU/内存条 + 玩家/TPS)+ **呼吸灯** | 启停/重启按钮 |
| **配置/记录** | 用户/组/模板/备份/告警/定时… | 列表行(可切卡片) | **属性字段**(无资源条),如告警 = 条件 code + 级别 + 最近触发 | 启用**开关 toggle** |

- **工作台卡**(运行实体):借鉴宝塔 aaPanel 工作台的"容器卡内嵌资源",换靛蓝大圆角 + iOS 丝滑 + 呼吸灯,去掉其方正小圆角/绿配灰/密集老派感。
- **配置行**(记录类):卡片化灵动行 + toggle 开关 + code 条件 + 级别 pill。
- 两者都支持**卡片 ⇄ 列表视图切换**;列表骨架同时作独立页 + §4.4 分组树的右侧面板。

## 6. 页面分类与排版优先级(2026-06-26 研究全 162 FR + 路由后产出)

### 6.1 五大呈现类型
- **监控仪表**:总览、平台/节点/实例监控 → 仪表骨架(环形 / 趋势 / 进程 tooltip / brush)。
- **运行实体**:实例、节点、Bot → 工作台卡(资源内嵌 + 呼吸灯),卡 ⇄ 列表。
- **配置记录**:告警、模板、定时、备份、备份存储、用户、组、网络、运行时资源、客户端分发 → 列表行(属性 + 开关 toggle)。
- **流水检索**:日志、审计 → 时间线 + 强筛选(日志可实时跟随)。
- **系统/工具/向导**:平台信息、设置、数据库、存储、系统更新、许可(信息/表单);实例工作区(全屏工具,从实例进);登录、初始化(向导)。
- **业务定制(JBIS)**:经济 / 背包 / 业务掌控台 → manifest 驱动,**特殊,不套通用范式**。

### 6.2 频率优先级(决定排版投入与导航位置)
- 🔥 **高频**:实例、总览、实例工作区、节点、日志(排查时)。
- ◐ **中频**:Bot、玩家、各监控页、告警、备份、群组。
- ○ **低频**:模板、定时、用户/组、备份存储、运行时资源、客户端分发、设置、数据库、存储、系统更新、许可、审计。

### 6.3 设计策略
1. 导航按频率排序,低频收进折叠组(承 FR-112)。
2. 默认视图按类型(运行=卡片 / 配置=列表 / 监控=仪表),可切换。
3. 密度按频率分级:高频"一屏速扫"密集,低频舒展清晰、可向导式。
4. 首屏聚焦"不正常的":运行类顶崩溃/高负载,配置类顶已启用/最近触发/最近改动。
5. 配置 vs 运行交互别混:运行=启停/重启按钮 + 呼吸灯 + 资源条;配置=启用开关 toggle + 无资源条。
6. 工具类不占顶级导航(从实例工作区进),保持顶级导航清爽。

## 7. 导航信息架构(2026-06-26 用户认可 5 域大方向)

从**当前 11 个一级**(粒度不一、单功能裸露)精简为 **5 个一级**,粒度统一、按运维心智分域:

1. **总览**(单)— 落地仪表盘。
2. **集群** — 节点 / 实例(多级分组) / 群组·拓扑。(你管的基础设施)
3. **监控** — 平台/节点/实例监控 / 告警 / 日志。(看状态和问题,告警/日志归此)
4. **运营** — 玩家 / Bot / 模板 / 备份(含存储) / 定时任务。(日常运营与任务;原裸露一级的单功能归此)
5. **系统**(低频沉底,内分两小节)— 「平台与维护」(平台/运行时/客户端分发/数据库/存储/更新) + 「账户与审计」(用户组/设置/审计/许可)。

- 高频域(集群/监控)在上,低频(系统)沉底;承接 FR-112(导航 IA 统一)、FR-131(侧栏可折叠图标轨)。
- **外壳**:5 域侧栏(当前域高亮 + 二级展开)+ 顶栏(面包屑 + 搜索 + 集群徽标 + 告警 + 账户)+ 侧栏底部**全局主题切换**;内容区按页面类型套范式(§4 / §6.1),换页面外壳不变。
- **归类已定**(2026-06-26 用户确认):Bot、模板归「运营」;「监控」独立成一级(即上述 5 域)。

## 8. 竞品研究与借鉴(2026-06-26,实看 Pterodactyl + MCSManager v10 截图)

### 标杆
- **MCSManager v10**(实看):已**浅色现代化**——实例**卡片网格**(状态+类型+标签徽标+操作图标)、清爽终端页、**应用市场(游戏封面 + 一键 Install + 环境需求)**;**验证「浅色卡片化」方向正确**。短板:仍偏功能化(终端页底部堆功能入口卡)、实例卡**无实时资源条**(本项目工作台卡更优)、精致/灵动不及本项目。
- **Pterodactyl**(实看):深色专业、**tab 式单服详情**(Console/Files/Databases/Schedules/Backups/Network/Startup/Settings)、资源/权限精细;短板 = 部署复杂、纯深色冷、偏单服。
- Pelican(Pterodactyl 现代重写,Filament):更干净现代但偏管理后台味。AMP:商业模块化漂亮但闭源收费。Aternos:托管极简友好但功能受限。

### 学过来(补强)
- **应用市场 / 一键开服**(首抄项,借 MCSM):把「模板」升级成带封面的应用市场——封面 + 类型/版本 + Java/RAM 需求 + 一键部署,靛蓝圆角灵动版**已出预览**,落 **FR-154**。
- **工作区补两个 tab**:**启动 Startup**(结构化启动可视化,JDK+JVM参数+jar+派生命令,ADR-008)+ **端口 Network**(端口分配,FR-032)。
- 工作区默认进「终端 + 资源」一屏掌控(Pterodactyl Console 式)。

### 差异化护城河(两家都弱,放大)
- 规模化:1000+ 多级嵌套分组、群组服拓扑(proxy↔backend,FR-145)、集群级监控。
- 独有能力:玩家治理、Bot 规模化、业务对接(经济/背包 JBIS)。
- 工程/体验:单二进制部署、靛蓝圆角灵动 + 双主题。

## 9. 工作区升级:可组合卡片画布 + 超级工作台 + 导播台(2026-06-26 用户核心创新)

**取代固定 tab(超越 FR-129 分屏),是最大差异化卖点——MCSM/Pterodactyl 均为固定单实例 tab,无此能力。**

- **卡片** = 实例 × 功能(终端 / 文件 / 配置 / 插件 / 状态 / Bot 各为一卡)。
- **画布** = 可拖拽网格,卡片自由摆放 / 调大小;卡头 = grip 拖拽手柄 + 实例·功能 + 全屏 / 关闭。
- **作用域**:① 单实例工作台(卡片限当前实例);② **超级工作台**(跨实例,如 4 个不同实例终端拼监看墙)。
- **预设** = 命名保存的画布布局(**个人级持久化**);原「终端/文件/配置」tab 降级为**快捷预设**(新手默认布局,高手自由拼)。
- **导播台** = 多预设**预热**(后台保活 WS 连接)+ 缩略图一排,点击 / 快捷键**瞬切零延迟** + 定时轮播(像 OBS 场景切换)。
- **技术核心 / 约束**:导播台预热 = 多实例 WS 连接保活;**必须设并发上限**,非当前场景卡片降频 / 暂停渲染,切换才全速,防浏览器过载。前端用可拖拽网格(react-grid-layout / gridstack 类)。
- **添加卡片 = 实例库面板拖拽**:左侧可收起「实例库」(搜索 + 实例可展开看 6 功能);**拖实例** = 加默认卡组,**拖功能** = 加单卡,**多选批量拖**(一次拼监看墙)。放置区高亮 + 松手落位。
- **单实例工作区** = 同画布、卡片限当前实例,默认「运维台」布局(大终端 + 状态 + 文件);「添加卡片」用同库、筛当前实例;原 tab → 快捷预设。
- **入口**:超级工作台作「集群」域独立入口,或从导播台进。
- **超出现有 FR,需登记新 FR**(可组合工作台 + 跨实例超级工作台 + 导播台)。
