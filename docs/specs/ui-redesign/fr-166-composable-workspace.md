# 功能规格：FR-166 可组合卡片工作区（单实例）

> 状态：草拟（待审核）　·　关联 PRD：FR-166（并入 FR-129/130）　·　依赖：FR-163/164（已 master）　·　批 3 第一批 / worktree B
> 关联 ADR：`ADR-034`（可组合卡片工作区，**取代 ADR-030**，占位号落地统一分配）

## 1. 背景与目标
design §9 的**最大差异化卖点**。实例工作区从「固定六 Tab（一次看一个）」演进为**可拖拽卡片画布**：卡片 = 实例 × 功能，自由摆放 / 调大小，命名预设保存布局。取代 ADR-030 的 `react-resizable-panels` 固定分屏方向，并为 FR-167（跨实例超级工作台）/ FR-168（导播台）铺路。本 FR 限**单实例**作用域。P2。

## 2. 需求（要什么）
### 范围内
- **可拖拽网格画布**：用 `react-grid-layout`（成熟、React 友好；spec 审核可改 `gridstack`）。卡片可拖拽摆放 / 调大小 / 不重叠流式布局。
- **卡片 = 实例 × 功能**：终端 / 资源（文件+配置合一，**承 FR-130**）/ 配置 / 插件 / 状态(监控) / Bot 各为一卡。卡头 = grip 拖拽手柄 + 实例·功能标签 + 全屏 / 关闭。
- **预设**：命名保存的画布布局（**个人级持久化**，先 `localStorage`，后端同步留后续）。原「终端/文件/配置」固定 tab **降级为快捷预设**（新手默认「运维台」布局 = 大终端 + 状态 + 资源）。
- **惰性挂载**：非激活 / 未打开的卡不预渲染（现状六 Tab 全挂载会同时建 WS + discover）——承 ADR-030 后果点。
- **文件 + 配置合一**（FR-130 并入）：资源卡复用既有 `ResourceExplorer`，配置发现/收藏作为该卡可选视图。

### 不做（范围外）
- 跨实例卡片拼合 / 实例库拖拽（FR-167）。
- 导播台多预设预热 / 瞬切 / 轮播（FR-168）。
- 后端预设同步（个人级先走 localStorage）。

## 3. 设计（怎么做）
- **新前端依赖**：`react-grid-layout`（拖拽网格）。无则引入；不引第二个拖拽库。
- **画布状态**（Zustand，`console.ts` 扩或新 `workspace.ts`）：卡片树（哪些卡、各卡 layout {x,y,w,h}、各卡打开的资源/功能 + 实例 id）；各卡**自管 dirty / 草稿**（承 ADR-030 状态外提）。预设 = 命名的 layout 快照，存 localStorage（纯函数 `lib/workspace-preset.ts` 序列化/校验，可测）。
- **卡片组件**：复用既有面板——终端（`TerminalPane`）、资源（`ResourceExplorer`，文件+配置合一）、插件（`PluginManager`）、监控（FR-060 历史曲线段）、Bot、状态。统一卡壳（grip 头 + 全屏 + 关闭）。
- **惰性挂载**：卡未在画布则不挂载；全屏 = 临时最大化单卡。
- **可寻址**（承 ADR-031）：当前预设 / 打开的卡可进 URL（刷新/分享复现），不强求本 FR 全量。
- 取代 ADR-030：写 `ADR-034`，标 ADR-030 `superseded-by`；FR-129/130 并入本 FR（PRD 置 ❌ 已废弃@并入FR-166）。
