# 功能规格：FR-168 工作区导播台

> 状态：草拟（主控自审）　·　关联 PRD：FR-168　·　依赖：**FR-167（待第二批落 master）**、FR-166　·　批 3 第三批
> 关联 ADR：`ADR-035`（导播台 WS 预热并发模型，新增）

## 1. 背景与目标
design §9 工作区链的收尾、最复杂一环。在 FR-166/167 的画布 + 跨实例预设之上，做**导播台**：多预设**预热**（后台保活 WS 连接）+ 缩略图一排 + 点击/快捷键**瞬切零延迟** + 定时轮播（像 OBS 场景切换）。**技术核心 = 多实例 WS 连接保活 + 并发上限 + 非激活降频**，防浏览器过载。P2。

## 2. 需求（要什么）
### 范围内
- **多预设预热**：选若干预设（场景）后台**保活**其 WS 连接（终端/监控等），使切换零延迟。
- **缩略图条 + 瞬切**：一排场景缩略图，点击 / 快捷键即时切到该场景（已预热 = 零延迟全速渲染）。
- **定时轮播**：可设间隔自动轮播场景（OBS 式）。
- **并发上限（硬约束）**：预热的 WS 连接数设上限（可配，默认 N）；超限按 LRU 驱逐最久未用的预热场景（降级为未预热，切换时重连）。
- **非激活降频 / 暂停渲染**：仅当前激活场景全速渲染；预热但非激活的场景**降频 / 暂停渲染**（WS 保活但不全速重绘），切到才全速。防多实例同时全速渲染过载。

### 不做（范围外）
- 后端改动（纯前端 WS 连接管理 + 渲染节流）。
- 跨设备同步 / 导播台预设云端持久。

## 3. 设计（怎么做，见 ADR-035）
- **导播台状态机**（`stores/workspace.ts` 扩或新 `stores/director.ts`）：场景（= FR-167 预设）× 状态 {激活(全速) / 预热(保活+降频) / 未预热}。激活唯一；预热集合受**并发上限**约束（LRU 驱逐）。
- **WS 连接池**：导播台管理多场景的 WS 保活（复用既有终端/监控 WS hook，加「保活但降频」模式）；非激活场景的卡用 `requestAnimationFrame` 节流 / `visibility` 暂停重绘（终端 xterm 不刷、监控不轮询或低频）。
- **瞬切**：切换 = 把目标场景置激活（已预热则零延迟）、原激活降为预热；快捷键（数字键/方向）。
- **轮播**：定时器按序切换激活场景（复用瞬切逻辑）。
- 纯逻辑下沉 `lib/director.ts`（并发上限 LRU 驱逐、轮播序列、场景状态转移）+ vitest 红→绿。
- 入口：导播台作超级工作台的一个模式 / 独立入口（design §9「超级工作台作集群域独立入口，或从导播台进」）。
- i18n 中/英；视觉复用卡壳 + 双主题。

## 5. 验收标准
- 多场景预热后点击 / 快捷键**零延迟瞬切**；定时轮播按序切换。
- **并发达上限时按 LRU 驱逐**最久未用预热场景，不无限建 WS；**非激活场景降频 / 暂停渲染**，浏览器不过载（多连接压测 CPU / 内存可控）。
- 切到某场景才全速；保活的 WS 不断（瞬切无重连等待）。
- i18n 中/英；暗/亮 + 双主题。**真机多连接压测由用户确认**（多场景预热 + 瞬切零延迟 + 轮播 + 上限降频不崩）。

## 6. 风险 / 待定
- 多 WS 保活的浏览器连接上限（HTTP/1.1 域名 6 连接 → WS 也受限）：并发上限默认值要保守；超限 LRU 驱逐。
- xterm / 监控的「降频但保活」实现（xterm 可停 render 但 WS 收数据缓冲；监控降轮询频率）——需逐组件确认可暂停渲染而不丢数据。
- 真机压测是本 FR 的硬验收维度（单元只能覆盖状态机逻辑，连接/渲染过载只能真机验）。
