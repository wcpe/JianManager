# 实施计划 — FR-093 发布/拉取全链路审计与追踪

> 关联 FR: FR-093 | 优先级: P1 | 状态: 🔨 in-progress | 关联 ADR: ADR-023
> 依赖: FR-092（机器码）。**发布事件审计已在 FR-087/088 落地**（`client_file.publish`/`client_version.publish`/`client_version.rollback` 入 audit），本 FR 补**拉取/下载事件追踪 + 数据量治理 + 检索**。

## 设计要点

### 数据量治理：明细短保留 + 写时增量聚合长保留 + 清理
- `client_dist_event`（**明细，短保留**）：每次 manifest 拉取 / 制品下载一行——channel/machine/ip/kind/version/artifact_sha/bytes/status/duration_ms/created_at。供按 IP/机器码/频道/版本/时间检索 + 近窗分布（去重机器码/IP）。
- `client_dist_daily`（**聚合，长保留**）：`(day, channel_id, version, kind)` UNIQUE，写时 upsert `requests+1`、`bytes+=N`。供下载量趋势 + 版本分布长期看（FR-095）。
- **清理 job**：后台周期删 `created_at` 早于保留期（默认 14 天）的明细行；聚合长留。复用 MetricService 式后台循环（FR-060）。
- 写入 best-effort、弱一致：失败仅日志，**绝不阻断玩家拉取**（与 FR-092 机器码登记同路径同原则）。

### 记录点
- `GET manifest`：鉴权后记 kind=manifest、version=manifest.version、bytes=响应字节、status、duration、ip、machine。304 也记（status=304、bytes=0）。
- `GET artifact`：鉴权后记 kind=artifact、artifact_sha、bytes=asset.Size、status、ip、machine。

### 检索（管理面）
- `GET /client-dist/events`（JWT 平台管理员）：按 channelId/machineId/ip/version/kind/since/until + limit 过滤，DESC 返回明细。
