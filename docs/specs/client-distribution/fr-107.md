# 实施计划 — FR-107 后台客户端更新器接入指引

> 关联 FR: FR-107 | 优先级: P1 | 状态: 🔨 in-progress（实现+真机验证完成，待用户确认） | 关联 ADR: ADR-021、ADR-022

## 背景

运营方在 JM 后台管理客户端分发频道，但不知道怎么把 OTA 更新器（wedge + updater-core 两件套 jar + jm-updater.json + 启动器 `-javaagent` 参数）接入整合包并下发玩家。后台需一页拿齐、照做即可接入。

## 设计要点（已与用户敲定）

- **形式/位置**：「客户端分发 → 频道详情」加「接入指引」Tab（频道专属、可复制现成配置）。
- **CP 内嵌两 jar**：`go:embed`（仿 `internal/controlplane/embed/probe.go`：目录 + `.gitignore` 占位 + make 注入），指引页提供下载按钮。
- **内容**：① 机制简述（fail-static/fail-open 绝不挡启动）② 下载 wedge.jar/updater-core.jar ③ 放置到 `<.minecraft>/jm-updater/` ④ **频道专属 jm-updater.json**（channel=本频道、endpoint 默认当前 origin+/api/v1 可编辑、密钥占位）⑤ 启动器 `-javaagent:jm-updater\wedge.jar`（相对路径推荐 + 绝对备选）⑥ 行为说明（fail-static/fail-open/FR-099 进度窗/与 authlib-injector 共存）。
- **边界**：纯运营面（面向运营方、非玩家）；不改 OTA 协议/manifest/客户端 jar 本身（只内嵌分发 + 引导展示）；jar 下载走 **JWT admin**（管理面，按 architecture-invariants 不用拉取密钥）。

## 实现落点

- **后端 embed**：`internal/controlplane/embed/client_updater.go`（`//go:embed all:client-updater` + `WedgeJar()`/`UpdaterCoreJar()` 访问器 + `ClientUpdaterEmbeddedVersion` 常量）；`embed/client-updater/.gitignore` 占位（jar 不入库）。
- **后端端点**：`router/client_updater_jars.go`（`ClientUpdaterJarsHandler`）——`GET /client-dist/updater-jars`（Info：版本+可用性）、`GET /client-dist/updater-jars/:component`（下载，`component∈wedge|core`，`Content-Disposition`；缺 jar 404）；`requirePlatformAdmin` 守卫；router.go admin 组无条件注册。
- **构建**：`Makefile` `embed-client-updater` 目标（`./gradlew :wedge:jar :updater-core:jar` → 拷进 embed 目录）。
- **前端**：`components/ClientIntegrationGuide.tsx`（独立组件，仿 ClientVersionsPanel/ClientStatsPanel）；`ClientChannelsPage.tsx` 频道详情加 Tab；`api/clientChannels.ts` 加 `useUpdaterJarsInfo()`/`downloadUpdaterJar()`；i18n `clientGuide.*` zh/en。

## 不做（范围边界）
- 不改 OTA 协议/manifest/客户端 jar；不做玩家面（仅运营面）；jar 下载不走拉取密钥（管理面 JWT）。
- 不做整包打包/CDN 配置（归运营方）。
