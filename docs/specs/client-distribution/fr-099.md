# 实施计划 — FR-099 客户端 OTA 更新进度窗口（进度条 + 速度 + ETA）

> 关联 FR: FR-099 | 优先级: P1 | 状态: 🔨 in-progress | 关联 ADR: ADR-021（纯 JVM、零依赖、Java 8）

## 背景

OTA 更新在 wedge `premain` 同步执行（`net.minecraft.client.main.Main` 之前，MC 渲染线程/LWJGL 尚不存在），玩家此刻只见启动器文本日志；大整合包首次/大更新时像卡死、无任何进度反馈。

**因 premain 早于 MC**，无法注入 MC 自身加载画面（更新结束 MC 才启动）。故由 updater-core 在更新期**自弹独立 Swing 进度窗口**展示进度，下载完自动关闭再放行 MC。

## 设计要点

### 展示形式（已与用户敲定）
- 独立 **Swing/AWT** 窗口（JDK 自带、零依赖、Java 8 可用、与启动器无关）。
- 展示：总体**进度条(0→100%)** + **当前速度(MB/s)** + **ETA(剩余秒)** + **当前文件名**。
- **仅在确有内容要下载时弹窗**（已是最新秒进、不闪空窗）——窗口在首个下载文件 `beginFile` 时惰性显示。
- 下载完成**自动关闭**；**headless**（`GraphicsEnvironment.isHeadless()`）自动降级**文本进度**（日志节流行），不报错、不阻断。
- 玩家**手动关窗 = 取消下载、fail-static** 以本地版本放行；窗口任意异常**吞掉降级**（保持 wedge fail-open / core fail-static，绝不挡启动）。
- 文案 **i18n（zh/en）** 随 `user.language`（core 侧自带小表，同 wedge `Messages` 风格）。

### 覆盖范围（按真实下载路径）
- ✅ **reconcile 逐文件下载**（FR-090，主场景，大下载在此）。
- ✅ **core 自更新 jar 下载**（FR-091 `SelfUpdater`）。
- ⛔ `.jmpack`（FR-097）**不在客户端 reconcile 下载路径**（`JmPack` 仅服务端打包 + 独立解包工具，未被 `Updater/Reconciler/SelfUpdater` 引用）→ 不涉及，不杜撰覆盖。

### 进度计量
- **分母（总字节）**：reconcile 前**廉价预估**（不哈希）——遍历适配文件，`!once-exists && !size匹配 && !cas.has(sha256)` 者计 `artifactSize`（压缩下载大小）之和；core 自更新另加 `AgentArtifact.size`。极少数「同大小不同 md5」误判由收尾 snap-to-100% 兜底。
- **分子（已下字节）**：下载在 `HttpTransport` 的 8192 分块读循环按块回调累加 → 字节级、实时（大文件不卡条）。
- **速度**：滑动窗口（~3s）样本 `(t, doneBytes)` 端点差分；**ETA** = 剩余/速度。纯逻辑、可注入时间戳单测。

### 落点与协议
- 主体在 **updater-core**；新增 `ProgressModel`（纯逻辑）/`ProgressView`(Swing|Text|Noop)/`ProgressReporter`/`CoreMessages`。
- `Transport` 加**默认方法** `fetchArtifact(sha, LongConsumer onBytes)`（默认委托旧法、末尾报总量；`HttpTransport` 重写为分块流式回调）——测试替身 `MemoryTransport` 不改（用默认）。
- `Updater` 创建 reporter，传 `Reconciler`/`SelfUpdater`，`finally` 关闭；可经 ctx `progressUi=false` 强制关。
- **不改服务端/manifest/楔子↔core §6.3 协议**（进度纯客户端算，core 已从 manifest 知各文件 size）。

## 不做（范围边界）
- 不注入 MC 加载画面（与 premain 时序矛盾）；不做暂停/重试/取消按钮（仅纯进度 + 关窗=fail-static）。
- 不改服务端/manifest/协议；不覆盖 `.jmpack`（不在客户端下载路径）。
- 不做品牌素材（标题文字即可）。

## 实施记录（2026-06-24）

- 实现完成：`ProgressModel`/`CoreMessages`/`ProgressView`(Swing/Text/Noop)/`ProgressReporter` 新增；`Transport` 默认方法 + `HttpTransport` 8192 分块流式回调；`Reconciler`（廉价预估 + 下载分支接 sink + 关窗取消）、`SelfUpdater`（core jar 下载接进度）、`Updater`（装配/关闭 reporter）、`Core`（ctx `progressUi` 开关）接入。
- **真机发现并修复 1 个 bug（类加载竞态）**：`SwingProgressView` 用 `invokeLater` 异步建窗，楔子 `CoreLoader` 在 `Core.run` 返回后关闭 core 的 `URLClassLoader`，EDT 异步 build 懒加载匿名内部类（`WindowAdapter`）时 classloader 已关 → `NoClassDefFoundError`、窗口建不出。改 `invokeAndWait` 同步化，类加载在 run 返回前完成。
- 测试：updater-core 全量绿（既有 0 回归 + 16 新进度单测：`ProgressModelTest`/`CoreMessagesTest`/`ProgressViewTest`）；test 任务设 `-Djava.awt.headless=true`。
- 真机证据：① PATH **真 Java 8(1.8.0_422)** headless 文本路径，rel099 36MB/6 文件端到端——文本进度行 + 6 文件全下载 + 无 `UnsupportedClassVersionError` + 游戏放行 + 无弹窗；② 非 headless Swing 路径修复后 `NoClassDefFoundError` 消失、窗口弹出（关窗触发取消→fail-static、downloaded=5/6、游戏放行）、JVM 干净退出。
- 待用户：PCL2 GUI 启动全屏可视确认（与相对路径 GUI 一并）；通过后由用户确认 FR-099 done。
