# 实施计划 — FR-095 分发统计后台

> 关联 FR: FR-095 | 优先级: P2 | 状态: 🔨 in-progress | 关联 ADR: ADR-023 | 依赖: FR-093、FR-094
> 复用 FR-093/094 既有聚合表（`client_dist_daily`/`client_telemetry_daily`）+ 明细（`client_dist_events`）+ FR-092 `client_machines`，**只读聚合，不引入新表**。

## 设计要点（只读聚合端点 + 前端看板）
- **`GET /client-dist/stats?channelId=&days=`（JWT 平台管理员）** → 复合统计：
  - **下载量趋势** ← `client_dist_daily` 按 day 汇总 requests/bytes。
  - **版本分布** ← `client_dist_daily`（kind=manifest）按 version 汇总 requests。
  - **更新成功率/回退率 + 结果分布** ← `client_telemetry_daily` 按 result 汇总（success/fail-static/rolled-back/error），算 successRate/rollbackRate。
  - **活跃机器码数** ← `client_dist_events` 近窗 `COUNT(DISTINCT machine_id)`。
  - **来源 IP 分布（Top N）** ← `client_dist_events` 近窗按 ip 计数 Top 10。
  - 按 `channelId` + `days`（默认 30）筛选；纯只读 GROUP BY，无新表。
- **前端**：频道详情新增「统计」Tab `ClientStatsPanel`——天数选择 + 下载趋势（复用 `TimeSeriesChart`）+ 版本分布（bar）+ 成功率/回退率 + 活跃机器码（数字卡）+ Top IP（表）。i18n（zh/en）+ 暗/亮主题（chart 用主题 token）。
