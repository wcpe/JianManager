# 实施计划 — FR-092 客户端机器码身份（仅追踪/统计）

> 关联 FR: FR-092 | 优先级: P1 | 状态: 🔨 in-progress | 关联 ADR: ADR-023（端点防护：限流以 IP 为主、机器码为辅）
> 依赖: 无（提供机器码身份，被 FR-093/094/095/096 复用）

## 背景

updater 拉 manifest/制品时契约已留 `X-Machine-Id` 头（HttpTransport 已发），但 core 此前传空（`Core.java` TODO）。
FR-092 让 updater 生成稳定、跨平台、不可逆的机器码并携带，服务端最小登记以供后续追踪/统计/辅助限流。
**机器码客户端生成、不可信**——仅统计 + 辅助限流，**不作信任/授权依据；限流以 IP 为主**（ADR-023）。

## 设计要点

### 客户端机器码生成（updater-core `MachineId`）
- **多硬件/环境特征组合 hash**：非回环 NIC 的 MAC（排序去重）+ `os.name`/`os.arch` + 主机名 + `user.name` + 逻辑 CPU 数 + 物理内存总量（`com.sun.management.OperatingSystemMXBean#getTotalMemorySize`）。
- **SHA-256 → 64 hex**：不可逆，**不暴露原始硬件信息**（仅哈希）。
- **稳定 + 部分变化容错**：首次计算后**持久化**到 `<userHome>/.jm-updater/machine-id`（per-machine，跨 gameDir/实例稳定）；后续直接读持久值——硬件部分变化（换网卡/插 U 盘）不改变身份。userHome 不可写则退化为每次按特征计算（仍尽量稳定）。
- **全程 best-effort、绝不抛**：任一特征采集失败即跳过；极端兜底返回固定特征的 hash。不引外部依赖、不 shell 外部命令（避免拖慢游戏启动 / 权限问题；主板/磁盘序列号需 shell，刻意不采，以持久化保稳定）。
- 接线：`Core.run` 以 `MachineId.get()` 取代传空，喂 `HttpTransport`（X-Machine-Id 已在请求头发出）。

### 服务端最小登记（control-plane）
- 表 `client_machines`：`channel_id` + `machine_id`（UNIQUE 组合）+ `first_seen` + `last_seen` + `hit_count`。
- `ClientMachineService.Record(channelID, machineID)`：upsert——存在则 `hit_count+1` 且更新 `last_seen`，否则插入。**弱一致、best-effort**（失败仅日志、绝不阻断玩家拉取）。
- 接入 `GET /client-channels/:id/manifest`：鉴权通过后，若 `X-Machine-Id` 非空则 `Record`（与既有「每次拉取更新 key.last_used_at」同属拉取路径轻写，模式一致）。
- **重分析/留存/按机器码维度统计归 FR-093（审计追踪）/FR-095（统计后台）**；本 FR 只立「登记表 + 关联」。

### 隐私
- 机器码为**不可逆 SHA-256**，本地仅存哈希；服务端只存哈希 + 计数/时间。**遥测合规告知玩家归 FR-094**（遥测可关）。

## 验收映射（FR-092）

| 验收标准 | 实现 |
|---|---|
| 多硬件特征组合 hash、稳定（部分变化容错）、不可逆、跨平台 | `MachineId`：NIC/CPU/内存/host/os 组合 SHA-256 + userHome 持久化（容错）；纯 JDK 跨平台 |
| 随 manifest/制品/遥测携带；服务端登记并关联 | HttpTransport 已发 X-Machine-Id（manifest+制品）；`client_machines` upsert |
| 客户端生成不可信；统计+辅助限流；限流以 IP 为主 | 文档明示；服务端仅登记/计数不作授权；限流主键 IP 见 FR-096 |
| 隐私：不可逆 hash | SHA-256，仅存哈希；遥测告知归 FR-094 |

## 不做（范围边界）
- 主板/磁盘序列号采集（需 shell 外部命令、拖慢启动/权限敏感）→ 以多特征+持久化保稳定，不采。
- 按机器码维度的统计看板 → FR-095；全链路追踪/留存 → FR-093；限流接入 → FR-096；遥测合规开关 → FR-094。
