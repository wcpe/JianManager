# 实施计划 — FR-088 客户端版本发布 + latest 指针（历史 / 运营回滚 / 管理台版本页）

> 关联 FR: FR-088 | 优先级: P1 | 状态: 🔨 in-progress | 关联 ADR: ADR-022（信任 + 防降级）
> 依赖: FR-087（版本快照模型 `ClientVersion`、`PublishVersion`/`PublishFile`/`BuildManifest`、latest 指针已就绪）
> 契约: `contract.md`（manifest latest-only、防降级单调 version）；本文件补「管理台侧」版本编排端点与 UI。

## 背景

FR-087 已落「最小发布链路」：上传 client-file 制品（`POST .../files`，内容寻址去重）+ 发布版本（`POST .../versions`，
服务端单调递增 `version`、切 `ClientChannel.CurrentVersion` latest 指针）+ 即时组装签名 manifest（`GET .../manifest`）。
`ClientVersion` 行天然保留（无任何删除路径），历史已在库。

FR-088 在此之上补三件「运营侧」能力，**不改 manifest 契约、不动客户端线**：
1. **版本历史查询**（仅管理台 JWT，绝不向玩家暴露——玩家只认 latest，contract §2）。
2. **运营回滚**：选历史版本内容，**以更高 version 重发为新 latest**（保持单调，客户端按 §3 防降级正常前进、不被拒）。
3. **管理台版本页**：历史列表 + 版本详情（文件清单）查看 + 回滚（二次确认 FR-059）+ **完整发布向导**。

## 设计要点

### 回滚 = 重发旧内容（不下发更低号）
- `Rollback(channelID, sourceVersion)`：读源 `ClientVersion` 快照（须属本频道）→ 反序列化 files/managedDirs/agent →
  复用 `PublishVersion` 以 **新 version=max+1** 重写为新行 + 切 latest。
- 复用 `PublishVersion` 即复用其校验 / 单调递增 / 事务切指针；制品内容寻址且历史保留，旧 artifact 仍在，回滚不缺料。
- 防降级闭环：新 version 恒 > 旧 latest，客户端 `lastSeenVersion` 不拒（ADR-022 §3、contract §3）。

### 历史查询只在管理面
- 列表 `GET /client-channels/:id/versions`、详情 `GET /client-channels/:id/versions/:version`——**JWT 平台管理员**，
  挂 admin 组（与发布端点同组，鉴权与玩家拉取密钥物理隔离，ADR-022 §4 / contract §4）。
- 消费侧 `GET .../manifest` 仍仅返回 latest，**不新增任意版本拉取**。

### 发布向导（codec=none 自动元数据）
- manifest 文件条目需 **解压后** `sha256/md5/size`（强校验 / 快筛）+ **制品** `artifact.sha256/size/codec`（下载寻址）。
- 向导走 **codec=none**：操作员上传原始文件 → `PublishFile` 入库即算 sha256+md5+size（`Asset` 已存两者）→
  回传供向导**自动填充** file.{sha256,md5,size} 与 artifact.{sha256(=同值),size,codec=none}；操作员只设 path/sync/platform。
- **zstd 压缩打包不在本 FR**：压缩 + `.jmpack` 容器属 FR-097，块级 diff 属 FR-098。向导先产 codec=none 版本，压缩后续叠加。
- `PublishFile` 结果补 `md5` 字段（供向导填 file.md5）。

## 任务拆解

### 后端 service（`internal/controlplane/service/client_version.go`）
- [ ] `ClientFileResult` 增 `MD5`（codec=none 时 = file.md5），`PublishFile` 回填 `asset.MD5`。
- [ ] `ListVersions(channelID) ([]VersionSummary, error)`：版本号 DESC；项含 version/note/fileCount/createdBy/createdAt/isLatest（对比 `CurrentVersion`）；频道不存在 → `ErrChannelNotFound`。
- [ ] `GetVersionDetail(channelID, version) (*VersionDetail, error)`：含 meta + 解析后 files/managedDirs/agent + isLatest；不存在 → `ErrVersionNotFound`。

## 验收映射（FR-088）

| 验收标准 | 实现 |
|---|---|
| 发布：制品+managedDirs/sync 组版、单调号、切 latest+重签 | FR-087 既有 `PublishVersion`（+向导 UI 接入） |
| 服务端保留历史、不向客户端暴露历史/任意版本拉取 | `ClientVersion` 全保留；历史端点仅 admin；消费侧仍 latest-only |
| 运营回滚：选历史以更高号重发为 latest（不触发防降级） | `Rollback` 复用 `PublishVersion`，新号 > 旧 latest |
| 发布/回滚入审计 | `client_version.publish`（既有）+ `client_version.rollback`（新） |
| 管理台版本列表 + 发布/回滚（二次确认）+ i18n/主题 | 版本 Tab + 发布向导 + `DangerConfirm` |

## 不做（范围边界）

- zstd 压缩 / `.jmpack` 容器 → FR-097；块级 diff → FR-098。
- 客户端本地 N-1 回退（boot-success 回滚）→ FR-091（客户端执行端）。
- 版本列表分页 → 暂全量 DESC，未来量大再加（记 backlog）。
