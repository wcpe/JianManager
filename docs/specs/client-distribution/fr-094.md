# 实施计划 — FR-094 客户端遥测上报

> 关联 FR: FR-094 | 优先级: P2 | 状态: 🔨 in-progress | 关联 ADR: ADR-023 | 依赖: FR-092（机器码）
> 端点契约见 contract §4.3：`POST /client-telemetry`（X-Client-Key + X-Machine-Id，202，隐私可关）。

## 设计要点
- **服务端**：`client_telemetry`（明细，短保留 14 天 + 滚动清理）+ `client_telemetry_daily`（按 day/channel/result 写时增量聚合，长保留，供 FR-095 成功率/回退率趋势）。`ClientTelemetryService.Record/Cleanup/Start/Stop`（仿 FR-093）。端点 `POST /client-telemetry`：拉取密钥（VerifyAnyKey）鉴权 + X-Machine-Id，body 见契约，**202 Accepted、best-effort 落库不阻塞**。挂 FR-096 守卫子组（限流/黑白名单同消费端点）。
- **客户端**：`Core.run` 在 reconcile 后 best-effort `transport.postTelemetry`（失败静默）。隐私 **opt-out**：`jm-updater.json` `telemetry:false` → 不上报（默认 true）。字段：result(success/fail-static/rolled-back/error) / fromVersion / toVersion / os / javaVersion / launcher / durationMs / bootSuccess / error。fromVersion/toVersion 取 reconcile 前后 `StateStore.lastSeenVersion`；不收集敏感个人数据（仅环境粗粒度 + 不可逆机器码）。
- **隐私告知**：机器码不可逆、遥测仅环境/结果粗粒度、可关——文档与玩家提示告知（运营整包侧）。
