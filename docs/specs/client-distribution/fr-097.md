# 实施计划 — FR-097 自有 `.jmpack` 打包（压缩 + 签名）

> 关联 FR: FR-097 | 优先级: P1 | 状态: 🔨 in-progress | 关联 ADR: ADR-021、ADR-022
> 依赖: FR-087（制品库/签名）、FR-090（客户端解码/验签）。**首期不加密、不含块级 diff**（flags 预留位）。

## 格式（`.jmpack`，所有多字节整数大端）

```
[0:6]   magic "JMPACK"
[6]     formatVersion uint8 = 1
[7]     flags uint8（bit0=encrypted 预留, bit1=diff 预留；首期 0）
[8:12]  metaLen uint32
[12:12+metaLen] meta JSON(UTF-8)：{"files":[{path, sha256(解压后原始), size(解压后), codec(zstd|none), offset, clen(payload 内压缩段长)}]}
[payload] 各文件压缩段顺序拼接（= 复用已存制品字节，codec 即制品 codec）
[尾部签名] keyIdLen uint8 + keyId + sig(Ed25519, 64B)
```

**签名覆盖原始字节** `bytes[0 : payloadEnd]`（magic→payload 末）——**非 canonical JSON**：两端对**同一段原始字节**签/验，跨语言天然一致（无需像 manifest 那样 canonical）。`payloadEnd = 12 + metaLen + Σclen`。

## 设计要点
- **服务端打包复用已存制品**：`JmPackService.PackVersion(channel)` 取 latest 版本 files → 按 `artifact.sha256` 读制品库字节（codec 即制品 codec，FR-088 向导 none / 运营 zstd）→ 拼 payload + 写 meta（解压后 sha256/size + codec + offset/clen）→ Ed25519 签名 → **入制品库 type=client-pack**（内容寻址去重）。**服务端无需新增 zstd 编码器**（打包已压缩制品；zstd 压缩在上传侧/FR-098）。
- **客户端解包**：updater-core `JmPack.unpack(bytes, trustStore)` → 验 magic/版本 → **Ed25519 验签**（内置公钥，签名缺/错拒）→ 解析 meta → 逐文件切 payload[offset:offset+clen] → 按 codec 解压（zstd-jni / none）→ **sha256 校验解压后内容** → 返回 path→bytes。
- **底层成熟库**：zstd（制品已用 zstd-jni）+ Ed25519（Go crypto/ed25519、JDK EdDSA，复用 FR-087/090 信任根与 dev keys）。**不自研加密、首期不加密**。
- **与 FR-087/090 衔接**：`.jmpack` 作 type=client-pack 制品入库；客户端可解包。**不改现有 per-file 投递主路径**（per-file 仍主投递）；`.jmpack` 为就绪容器，供 FR-098 块级 diff（启用 diff flag）与可选打包投递复用。
- **跨语言兼容**：golden 向量——Go 打包产出固定 `.jmpack`（base64），Java 测试解包 + 验签 + 内容比对（仿 `ServerManifestCompatTest`）。

## 任务拆解
- [ ] Go `service/jmpack.go`：`PackJmPack(files, priv, keyID)`（写格式 + 签名）+ `ParseJmPack(data, pub)`（验签 + 解析，**不解压**，返回每文件压缩段 + meta）+ 常量。
- [ ] Go `service/jmpack_service.go`：`JmPackService.PackVersion(channelID)` 读 latest 版本 + 制品字节 → PackJmPack → 入库 type=client-pack；admin 端点 `POST /client-channels/:id/pack`（返回 .jmpack 制品 sha256，写审计）。

- [ ] Java 测试：golden 向量解包 + 验签 + 内容比对 + 篡改拒。

## 验收映射
| 验收 | 实现 |
|---|---|
| 容器格式 header(magic/版本/flags)+meta(路径/sha256/大小/算法)+payload(zstd)+签名；flags 预留加密/diff | 上述格式，flags bit0/bit1 预留 |
| 底层成熟库（zstd+Ed25519）；不自研加密、首期不加密 | zstd-jni/klauspost 无新增（打包已压缩制品）+ crypto/ed25519/JDK EdDSA |
| 服务端打包（压缩+签名入库）+ 客户端解包（验签+解压） | `JmPackService.PackVersion` 入库 + `JmPack.unpack` |
| 与 FR-087 制品、FR-090 reconcile 衔接 | type=client-pack 制品 + 客户端解包；不改主投递 |

## 不做
- 加密（flags 预留，首期不做）；块级 diff（FR-098，复用本容器 diff flag）；打包投递替换 per-file 主路径（保留 per-file）。
